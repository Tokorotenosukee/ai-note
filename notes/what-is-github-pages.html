GitHub Pagesとは何か

作成日: 2026-06-16

3行まとめ

  • GitHub Pagesは、GitHubリポジトリ内のHTML、CSS、JavaScriptなどをWebサイトとして公開する静的ホスティング機能です。
  • 個人サイト、プロジェクト紹介、ドキュメント、今回のAI Notesのような静的ナレッジベースに向いています。
  • 公開されたPagesサイトはインターネット上から見えるため、秘密情報や個人情報を置かない運用が重要です。

GitHub Pagesとは

GitHub Pagesは、GitHubが提供する静的サイトホスティング機能です。 GitHub公式ドキュメントでは、リポジトリ内のHTML、CSS、JavaScriptファイルをそのまま、またはビルド処理を通してWebサイトとして公開できるサービスと説明されています。

通常のWebアプリのようにサーバー側でデータベース処理をしたり、ユーザーごとにページを生成したりするものではありません。 あらかじめ置かれた静的ファイルを配信する仕組みなので、構成が単純で、個人サイトやドキュメント公開に向いています。

公開されるURL

GitHub Pagesには、大きく分けてユーザー・組織サイトとプロジェクトサイトがあります。 ユーザーサイトは通常 <owner>.github.io という名前のリポジトリから公開され、 プロジェクトサイトは各リポジトリごとに公開されます。

たとえば、このAI Notesのように ai-note というリポジトリをプロジェクトサイトとして公開する場合、 標準的なURLは https://tokorotenosukee.github.io/ai-note/ のようになります。 独自ドメインを設定することもできます。

どうやって公開するか

GitHub Pagesの公開元は、特定のブランチとフォルダに設定できます。 ビルド処理を細かく制御する必要がない場合、公式ドキュメントでは、特定ブランチへのpushをきっかけに公開する方法が案内されています。

このリポジトリのようにHTMLとCSSだけで構成している場合は、GitHubのリポジトリ設定から Pages を開き、Sourceを Deploy from a branch、 Branchを main、Folderを /(root) にするのが分かりやすいです。 その後、main にpushした内容がPagesサイトへ反映されます。

何に向いているか

GitHub Pagesは、更新頻度が高すぎず、静的ファイルとして配れる情報に向いています。 たとえば、以下のような用途です。

  • 個人プロフィールやポートフォリオ
  • OSSや小さなプロジェクトのドキュメント
  • ブログや読書ノートのような静的記事
  • HTML、CSS、画像だけで作る小規模サイト
  • AI Notesのような個人ナレッジベース

向いていないこと

GitHub Pagesは静的ホスティングなので、ログイン機能、会員ごとの非公開ページ、サーバー側の決済処理、 データベース更新を伴うアプリには向いていません。 GitHubの制限ページでも、パスワードやクレジットカード番号の送信のような機密性の高い取引には使うべきではないと説明されています。

また、GitHub Pagesは無料の汎用WebホスティングとしてオンラインビジネスやSaaSを運営する用途を想定したものではありません。 大量トラフィック、大容量ファイル、頻繁なビルドが必要な場合は、別のホスティングサービスを検討した方がよいです。

セキュリティと公開時の注意

Pagesサイトはインターネット上に公開されます。GitHub公式ドキュメントでも、 PagesサイトはリポジトリがPrivateであっても公開される場合があるため、機密情報があるなら公開前に削除するよう注意されています。

そのため、公開リポジトリでGitHub Pagesを使う場合は、APIキー、アクセストークン、住所、メールアドレス、 仕事の内部情報、非公開資料の内容を置かないことが重要です。 一度Gitにコミットした情報は履歴に残るため、後からファイルだけ消しても完全に消えたとは限りません。

HTTPSについては、GitHub PagesはHTTPSとHTTPS enforcementをサポートしています。 github.io ドメインのPagesサイトはHTTPSで提供されます。 ただし、HTML内で http:// の画像、CSS、JavaScriptを読み込むと混在コンテンツになり、安全性や表示に問題が出ることがあります。

AI Notesで使う意味

AI Notesでは、記事をHTMLファイルとして保存し、index.html からリンクするだけでサイト全体が成り立ちます。 GitHub Pagesと相性がよく、ビルドツールやCMSを使わなくても、GitHubへpushするだけで記事を公開できます。

その一方で、Public運用にするなら「公開してよい一般知識だけを書く」というルールが重要です。 個人的な日記や仕事の内部メモではなく、あとから自分も他人も読める調査記事として残す使い方が安全です。
